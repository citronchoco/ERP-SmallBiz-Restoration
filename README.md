# ERP-SmallBiz-Restoration

**프로젝트 개요**

본 프로젝트는 기존 ERP 시스템 도입에 어려움을 겪는 소상공인들을 지원하기 위해 개발된 무료 자원 관리 플랫폼입니다. 재고 관리 및 실시간 데이터 처리를 위한 백엔드 API 서버 로직을 포함하고 있습니다. 그룹 프로젝트로 진행되었으며, 현재 저장소는 프로젝트의 기술적 기록을 보존하기 위한 개인 아카이브입니다.

*아카이브 공지 (Project Restoration)*
- 상태: 원본 공유 저장소의 예기치 못한 데이터 유실 및 관리 기록 삭제로 인해, 본인이 로컬에 보유하고 있던 백업본을 바탕으로 복구한 저장소입니다.

- 복구 사유: 협업 과정에서 발생한 원본 저장소의 히스토리 소실 사고에도 불구하고, 프로젝트의 전체 구조와 본인이 기여한 데이터베이스 설계 및 백엔드 로직을 보호하기 위해 긴급히 아카이브를 구축했습니다.

- 구성: 본 아카이브는 프로젝트 전체의 연속성을 위해 프론트엔드와 백엔드 소스 코드를 모두 포함하고 있습니다.

### Tech Stack

Backend: Java, Spring Boot

Frontend: Vue.js (JavaScript, HTML, CSS)

Database: MySQL

Design Tool: MySQL Workbench (ERD 설계)

<br>

### 주요 기여 내용

1. 자원 관리 및 상태 자동화 엔진 설계

데이터 모델링 (Master-Transaction 분리): 자원 마스터(rm_resource)와 실시간 이용 내역(rm_usage)의 관계를 정규화하여 데이터 무결성을 확보하고 대여 이력을 체계화했습니다.

Time-driven 상태 자동 전환: CASE 문 기반의 단일 쿼리를 설계하여 현재 날짜 기준 '예약 ➜ 사용중 ➜ 사용 종료' 상태가 자동 갱신되는 Zero-maintenance 관리 프로세스를 구축했습니다.

시계열 히스토리 트래킹: 이용자 인적 사항과 이용 시점을 시계열 데이터로 보존하여 관리자 대시보드에서 전수 이력 조회가 가능하도록 구현했습니다.

2. 데이터베이스 최적화 및 예약 유효성 검증

Recursive CTE 기반 점유 검증: 재귀 쿼리(Recursive CTE)를 활용해 예약 기간을 일 단위 데이터로 동적 생성함으로써, 서버 사이드의 복잡한 로직 없이 DB 레벨에서 중복 예약을 차단했습니다.

수정/조회 예외 로직 설계: 예약 수정 시 본인 점유 데이터는 제외하고 중복을 체크하는 등 실제 운영 환경의 예외 케이스를 고려한 SQL 로직을 반영했습니다.

3. 위기 대응 및 기술 자산 보존

아카이브 구축 및 전수 복구: 공유 저장소 사고 시 로컬 백업 자산을 활용해 프로젝트를 복구하며 실무 환경의 데이터 백업 및 형상 관리의 중요성을 실천했습니다.
